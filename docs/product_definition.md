# LEXI — 나만의 AI 지식 사전
**Product Definition v1.0** | 2026.02 · 토이 프로젝트 (개인)

---

## 1. 문제 정의

| | |
|---|---|
| **페인 포인트** | 수업/논문에서 배운 개념이 한 학기 뒤 사라진다. '감각'은 남지만 언어로 꺼내지 못한다. |
| **근본 원인** | 기록 자체가 피로하다. 완성된 노트를 만들어야 한다는 부담이 진입장벽이 된다. |
| **기존 도구 한계** | Notion: 구조 잡는 부담. Obsidian: 연결 설정 피로. NotebookLM: 노트북 간 연결 없음. Recall: 내 언어/맥락 없이 키워드 연결. |
| **핵심 인사이트** | 기존 도구들은 모두 '내가 왜 이 개념에 꽂혔는지'를 저장하지 않는다. 연결의 기준이 키워드/문서가 아니라 내 이해의 맥락이어야 한다. |

---

## 2. 솔루션 컨셉

PDF를 던지면 → 내 말로 개념을 정착시키고 → AI와 짧은 대화로 깊이를 조절하고 → 자동으로 연결이 쌓이는 나만의 지식 사전.

**기존 PKM 도구와의 핵심 차별점**
- 연결의 기준 = 키워드/태그가 아닌 '내가 이해한 맥락'
- 입력 마찰 최소화 — PDF 올리고 한 줄만 써도 저장 완료
- AI가 연결 제안 시 '왜 연결됐는지' 설명 포함

---

## 3. MVP 플로우

| 단계 | 이름 | 설명 |
|---|---|---|
| ① | 입력 | PDF 업로드 (논문 / 강의교안) |
| ② | 추출 & 확인 | Gemini가 핵심 개념 추출 → 내가 확인 / 편집 |
| ③ | 인사이트 & 학습 대화 | '나는 이게 왜 대박이었나' 한 줄 입력 → Gemini가 방향 제안 (깊게 / 연결 / 저장) → 선택에 따라 1~5턴 대화 → 저장 확정 |
| ④ | 연결 제안 | 기존 카드들과 AI가 연결 제안 + '왜 연결됐는지' 한 줄 설명 표시 |
| ⑤ | 탐색 | 카드 목록 + 연결 그래프 보기 / 내 언어로 저장된 개념 검색 |

---

## 4. 화면 구성 (MVP 3-Screen)

| 화면 | 주요 기능 |
|---|---|
| ① 업로드 / 입력 | PDF 업로드 → 개념 추출 확인/편집 → 인사이트 입력 → 학습 대화 → 저장 |
| ② 카드 상세 | 저장된 개념 카드 보기 · 내 인사이트 + 대화 기록 · 연결된 카드 목록 (이유 포함) |
| ③ 그래프 탐색 | 전체 개념 카드 목록 · 연결 그래프 시각화 · 키워드 검색 |

## 5. 기술 스택

| | |
|---|---|
| **Frontend** | React 18 + Vite · Tailwind CSS · 바이브코딩 전용 |

## 6. 개발 원칙 (바이브코딩 태도)

- AI가 코드를 짜줘도, '이 로직이 왜 이렇게 됐지?'를 한 번은 물어본다
- MVP 완성 후 문제 정의 문서를 반드시 업데이트한다
- 기능 욕심이 생기면 이 문서로 돌아와서 범위를 점검한다
- 연결, 검색 외 신규 기능은 MVP 완성 이후에 논의한다
